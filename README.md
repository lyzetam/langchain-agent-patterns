## 🎯 What's covered

### LangChain (`create_agent`)
- ✅ Agent creation, tool definition & management
- ✅ Short-term & long-term memory / persistence
- ✅ Streaming & real-time updates
- ✅ Middleware (`@wrap_model_call`, `@dynamic_prompt`, `@wrap_tool_call`, HITL)
- ✅ Structured output

### LangGraph
- ✅ StateGraph API — nodes, edges, conditional routing
- ✅ Functional API (`@entrypoint` / `@task`)
- ✅ Map-reduce with `Send`, `Command` for route+update, subgraphs & `Command.PARENT`
- ✅ Checkpointing, durability, time-travel

### Deep Agents (`deepagents` package)
- ✅ `create_deep_agent` harness with built-in planning (`write_todos`)
- ✅ Virtual filesystem & context management
- ✅ Subagents, skills (on-demand `SKILL.md`), long-term memory
- ✅ Human-in-the-loop, pluggable backends

### Multi-agent systems (5 current patterns)
- ✅ **Subagents** (agent-as-tool) · **Handoffs** (`Command.PARENT`) · **Skills** · **Router** · **Custom workflow**

### LangSmith
- ✅ Tracing, evaluation, datasets & experiments, annotation queues

## 🚀 Quick start

### Install

```bash
pip install -U langchain "langchain[anthropic]" langgraph   # core
pip install -U deepagents                                   # for the Deep Agents example
pip install langgraph-checkpoint-sqlite                     # optional: durable persistence
```

### Environment

```bash
export ANTHROPIC_API_KEY="..."      # or OPENAI_API_KEY / GOOGLE_API_KEY, etc.

# Optional: LangSmith tracing
export LANGSMITH_TRACING=true
export LANGSMITH_API_KEY="..."
export LANGSMITH_PROJECT="my-project"
```

### Minimal agent

```python
from langchain.agents import create_agent
from langchain.tools import tool

@tool
def search(query: str) -> str:
    """Search for information."""
    return f"Results: {query}"

agent = create_agent(
    model="anthropic:claude-sonnet-4-6",
    tools=[search],
    system_prompt="You are a helpful assistant.",
)
print(agent.invoke({"messages": [{"role": "user", "content": "Hello!"}]})["messages"][-1].content)
```

### Run the Deep Agents sample

```bash
cd examples/release-notes-agent
pip install -e .                                  # or: uv sync
python agent.py                                   # bundled sample commits
python agent.py --repo /path/to/repo --version 1.2.0   # real git log
langgraph dev                                     # serve via langgraph.json
```

## 📖 Key concepts

### State & memory

```python
from langgraph.checkpoint.memory import InMemorySaver   # short-term (per thread)
from langgraph.store.memory import InMemoryStore        # long-term (across threads)

graph.compile(checkpointer=InMemorySaver(), store=InMemoryStore())
```

### Streaming

```python
for chunk in agent.stream(input, stream_mode="updates"):       # progress
    print(chunk)
for token, meta in agent.stream(input, stream_mode="messages"): # tokens
    print(token.content, end="")
```

### Multi-agent handoff (current API)

```python
from langchain.messages import AIMessage, ToolMessage
from langchain.tools import tool, ToolRuntime
from langgraph.types import Command

@tool
def transfer_to_sales(runtime: ToolRuntime) -> Command:
    """Hand control to the sales agent."""
    last_ai = next(m for m in reversed(runtime.state["messages"]) if isinstance(m, AIMessage))
    return Command(
        goto="sales_agent",
        graph=Command.PARENT,
        update={"active_agent": "sales_agent",
                "messages": [last_ai, ToolMessage("Transferred to sales", tool_call_id=runtime.tool_call_id)]},
    )
```

## 🏗️ Choosing an approach

```
Single agent, model + tools loop?             → create_agent          (basic_agent.py)
Batteries-included (plan/memory/subagents)?   → Deep Agents           (release-notes-agent/)
Explicit control flow / deterministic stages? → LangGraph             (langgraph_*.py)
Multiple specialized agents?
  ├── Centralized coordinator                 → Subagents             (supervisor_multi_agent.py)
  ├── Peer control transfer                   → Handoffs              (swarm_multi_agent.py)
  ├── One agent, swap expertise per task      → Skills
  ├── Classify then dispatch (parallel)       → Router
  └── Bespoke multi-stage pipeline            → Custom workflow
```

### Relative profile (qualitative — verify against your workload)

| Approach | Setup | Latency | Token cost | Parallel | Best for |
|----------|-------|---------|-----------|----------|----------|
| `create_agent` | Low | Medium | Medium | No | General single-agent tool use |
| Deep Agents | Low | Medium–Higher | Higher | Yes | Open-ended work: planning + memory + subagents |
| LangGraph | Medium–High | Varies | Varies | Yes | Explicit control flow, durability |
| Subagents | Medium | Higher | Higher | Yes | Multi-domain, centralized control, isolation |
| Handoffs | Medium | Medium | Medium | No | Conversational peer transfer |
| Router | Medium | Lower–Medium | Medium | Yes | Classify + parallel multi-source |

> These are relative, not benchmarks — actual cost/latency depends on model, prompt, and task.

## 🛠️ Best practices

- **Tools** — clear typed signatures + docstrings (the docstring is what the model reads).
- **State** — `add_messages` reducer for history; keep state minimal.
- **Multi-agent** — start with one agent; split only when isolation/parallelism justifies it.
- **Production** — always use a checkpointer, enable LangSmith tracing, set recursion limits, handle errors.
